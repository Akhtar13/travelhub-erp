# TravelHub ERP Development Guide

## Documentation Order

When working on this project, always read the files in this order:

1. crud.md
2. phaseX.md (the phase requested by the user)

Rules:

- Never implement future phases.
- Never change project architecture.
- Follow Laravel 12 best practices.
- Keep Controllers thin.
- Business logic belongs in Services.
- Use Form Requests for validation.
- Use Policies for authorization.
- Generate production-ready code.
